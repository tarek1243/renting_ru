Hosting guide

Three options — from easiest to most control

★ Recommended — easiest
Railway ~$5–15 / month

Connect your GitHub repo, click deploy. Railway auto-detects the Dockerfiles, adds Postgres + Redis plugins in one click, and wires the connection strings automatically. No server management.

✓ PostgreSQL plugin ✓ Redis plugin ✓ Auto-deploy on git push ✗ No built-in S3 → use Cloudflare R2 (free)
1
Push the repo to GitHub: git init && git add . && git commit -m "init" && git push
2
Go to railway.app → New Project → Deploy from GitHub → select your repo
3
Add database plugins: click + New → Database → PostgreSQL, then repeat for Redis
4
Add 3 services — one per app. Set the Dockerfile path for each:
API: apps/api/Dockerfile
Web: apps/web/Dockerfile
Admin: apps/admin/Dockerfile
5
Railway auto-injects DATABASE_URL and REDIS_URL. Add the remaining env vars to the API service:
JWT_ACCESS_SECRET = any long random string
JWT_REFRESH_SECRET = different long random string
ENCRYPTION_KEY = exactly 32 characters
6
On the Web and Admin services, set:
NEXT_PUBLIC_API_URL = the Railway URL of your API service
7
File uploads: create a free Cloudflare R2 bucket → add S3_ENDPOINT, S3_ACCESS_KEY, S3_SECRET_KEY, S3_BUCKET, S3_PUBLIC_URL to the API service
8
Seed the database: Railway dashboard → API service → Console tab → run:
pnpm --filter @renting/api migrate:deploy && pnpm --filter @renting/api seed
Good alternative
Render ~$7–20 / month

Similar to Railway. Connect GitHub, create Web Services for API + Web + Admin, add a Postgres database. Free tier available but sleeps after 15 min inactivity.

✓ PostgreSQL managed ✗ Redis costs extra → use Upstash (free tier) ✓ Auto-deploy on push ✗ Free tier sleeps after 15 min
1
Go to render.com → New → PostgreSQL → create database → copy the connection string
2
Get a free Redis from upstash.com → copy the REDIS_URL
3
New → Web Service → connect GitHub → Docker runtime → Dockerfile path: apps/api/Dockerfile
4
Add env vars: DATABASE_URL, REDIS_URL, JWT_ACCESS_SECRET, JWT_REFRESH_SECRET, ENCRYPTION_KEY (32 chars)
5
Repeat step 3 for Web (apps/web/Dockerfile) and Admin (apps/admin/Dockerfile)
6
On Web + Admin: set NEXT_PUBLIC_API_URL to your Render API URL
7
Render Shell on API service → run pnpm --filter @renting/api migrate:deploy && pnpm --filter @renting/api seed
Most control, cheapest long-term
VPS with Docker (Hetzner / DigitalOcean) ~$6 / month

Rent a VPS, clone the repo, run docker compose up -d. Works exactly like your local setup. Cheapest option but you manage the server yourself.

✓ Works exactly like local dev ✓ MinIO included — no separate S3 needed ✗ You manage SSL, updates, backups ✗ Basic Linux knowledge required
1
Create a VPS at Hetzner (cheapest), DigitalOcean, or Vultr — Ubuntu 22.04, 2 GB RAM minimum
2
SSH in and install Docker:
curl -fsSL https://get.docker.com | sh
3
Clone the repo:
git clone https://github.com/you/renting_ru && cd renting_ru
4
Copy and fill in env file:
cp .env.example .env && nano .env
Set strong values for JWT_ACCESS_SECRET, JWT_REFRESH_SECRET, ENCRYPTION_KEY
5
Start all services:
docker compose up -d
6
Migrate and seed:
docker compose exec api pnpm --filter @renting/api migrate:deploy
docker compose exec api pnpm --filter @renting/api seed
7
Add HTTPS with Caddy (easiest SSL) — point your domain to the server IP, Caddy handles the certificate automatically
All three options also need:
Email: Use Resend.com — 3,000 free emails/month. Set SMTP_HOST=smtp.resend.com, SMTP_USER=resend, SMTP_PASS=re_yourkey.
File storage (Railway/Render): Cloudflare R2 — 10 GB free, S3-compatible. Set S3_ENDPOINT, S3_BUCKET, S3_ACCESS_KEY, S3_SECRET_KEY, S3_PUBLIC_URL.
Domain: Buy from Cloudflare, Namecheap, or GoDaddy. Point DNS to your hosting provider.
